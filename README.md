## ◈ About

<p>
I'm a <strong>Full Stack Engineer</strong> and <strong>Co-Founder & CTO</strong> of <a href="https://kraevo.in">Kraevo</a> — a healthtech startup building intelligent department-wise queue management for Tier 2 and Tier 3 Indian hospitals, with native integrations for <strong>ABDM / Ayushman Bharat</strong> and AI-powered operational intelligence.
</p>

<p>
My engineering foundation is built on the <strong>PERN stack</strong> (PostgreSQL, Express, React, Node.js) with TypeScript-first development, Next.js for production-grade frontend systems, and Prisma ORM for type-safe data modelling. I build for scale, correctness, and developer experience — from zero to shipped.
</p>

<p>
Beyond the product, I'm deeply invested in the <strong>AI/ML engineering</strong> frontier: RAG pipelines, embedding-based semantic search, and the practical deployment of language models into real-world product surfaces. I run <strong>Arch Linux with Hyprland</strong> on a Wayland compositor — my environment is as intentionally engineered as my code.
</p>

<p>
I'm currently preparing for <strong>NIMCET</strong> with NIT Trichy as my target, pursuing remote <strong>AI engineering internships</strong>, and growing Kraevo from pilot-stage to market.
</p>

**Open To:**
- Remote AI / Full Stack Engineering Internships
- Open Source Collaboration — healthtech, developer tooling, AI infrastructure
- Technical advisory or co-building in early-stage startups

## ◈ Featured Projects

<details>
<summary><strong>Kraevo — Hospital Intelligence Platform</strong></summary>

<br/>

A B2B healthtech SaaS platform delivering department-wise intelligent queue management and operational analytics to Tier 2 and Tier 3 Indian hospitals. Built with full Ayushman Bharat / ABDM integration and designed for low-infrastructure environments.

| Attribute | Detail |
|:---|:---|
| **Stack** | Next.js · TypeScript · PostgreSQL · Supabase · Prisma ORM · Node.js |
| **Scale** | 11 multi-speciality hospital pilots · 2 confirmed production deployments |
| **Performance** | Sub-200ms queue update latency · real-time department dashboard |
| **Security** | DPDPA 2023 compliant · Supabase RLS · encrypted patient identifiers |
| **Impact** | Measurable reduction in OPD wait times · weekly doctor feedback loop |
| **Repository** | Private — stealth pre-seed stage |

Kraevo addresses a structural gap in Indian hospital operations: the absence of intelligent, affordable queue infrastructure accessible to facilities outside metro Tier 1 markets. The platform integrates government health scheme data pipelines (Ayushman Bharat / ABHA) and exposes a department-aware real-time queue engine. GTM strategy deploys free one-month pilots in exchange for structured clinical feedback, accelerating product-market fit validation with domain experts.

</details>

| Attribute | Detail |
|:---|:---|

| **Scale** | Multi-tenant architecture · role-based access (admin / instructor / student) |
| **Performance** | Lazy-loaded course modules · paginated content delivery |
| **Security** | JWT authentication · bcrypt password hashing · parameterised SQL |
| **Impact** | AI quiz generation reduces content creation time by ~70% |
| **Repository** | [github.com/Goofy-elf1](https://github.com/Goofy-elf1) |

The LMS integrates a language model backend for dynamic question generation from course material, adaptive difficulty adjustment, and personalised learning path suggestions. The architecture separates instructor and student contexts into distinct role surfaces, with a shared PostgreSQL schema modelled for multi-tenant isolation via Prisma.

</details>
